# Input Components

Proton provides a comprehensive set of input components for forms and data entry, including text inputs, checkboxes, radio buttons, and specialized inputs.

## Input

The base text input component.

**Location**: `components/input/Input.tsx`

<Note>
  This component is deprecated. Consider using `Input` from `@proton/atoms` or `InputFieldTwo` for new code.
</Note>

### Basic Usage

```tsx
import Input from '@proton/components/components/input/Input';
import { useState } from 'react';

const MyForm = () => {
  const [value, setValue] = useState('');

  return (
    <Input
      value={value}
      onChange={(e) => setValue(e.target.value)}
      placeholder="Enter text"
    />
  );
};
```

### Props

Extends `InputHTMLAttributes<HTMLInputElement>`

<ParamField path="value" type="string" required>
  The input value
</ParamField>

<ParamField path="onChange" type="(e: ChangeEvent) => void" required>
  Change event handler
</ParamField>

<ParamField path="placeholder" type="string">
  Placeholder text
</ParamField>

<ParamField path="error" type="string">
  Error message to display below the input
</ParamField>

<ParamField path="icon" type="ReactElement">
  Icon element to display on the right side
</ParamField>

<ParamField path="loading" type="boolean">
  Shows loading state and disables input. Default: `false`
</ParamField>

<ParamField path="fullWidth" type="boolean">
  Makes input take full width. Default: `true`
</ParamField>

<ParamField path="disabled" type="boolean">
  Disables the input
</ParamField>

<ParamField path="required" type="boolean">
  Marks field as required. Default: `false`
</ParamField>

<ParamField path="onPressEnter" type="KeyboardEventHandler">
  Handler called when Enter key is pressed
</ParamField>

## Checkbox

Checkbox input with label support.

**Location**: `components/input/Checkbox.tsx`

### Usage

```tsx
import Checkbox from '@proton/components/components/input/Checkbox';

const MyForm = () => {
  const [agreed, setAgreed] = useState(false);

  return (
    <Checkbox
      checked={agreed}
      onChange={(e) => setAgreed(e.target.checked)}
    >
      I agree to the terms and conditions
    </Checkbox>
  );
};
```

## Best Practices

### Form Validation

```tsx
const [email, setEmail] = useState('');
const [error, setError] = useState('');

const validateEmail = (value: string) => {
  if (!value) {
    setError('Email is required');
  } else if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value)) {
    setError('Invalid email format');
  } else {
    setError('');
  }
};

<Input
  type="email"
  value={email}
  onChange={(e) => {
    setEmail(e.target.value);
    validateEmail(e.target.value);
  }}
  error={error}
  required
/>
```

### Accessibility

```tsx
// Provide proper labels
<label htmlFor="username">Username</label>
<Input
  id="username"
  value={username}
  onChange={(e) => setUsername(e.target.value)}
  aria-describedby="username-help"
/>
<span id="username-help" className="text-sm color-weak">
  Choose a unique username
</span>
```

### Controlled Components

Always use controlled components with state:

```tsx
const [value, setValue] = useState('');

<Input
  value={value}
  onChange={(e) => setValue(e.target.value)}
/>
```
